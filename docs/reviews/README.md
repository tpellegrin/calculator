# Review artifacts

Canonical location for review reports that provide lasting value.

## When to commit a review report

- Ordinary task reviews: Findings may remain in task execution records or AI usage logs.
- Permanent review artifacts are appropriate for:
  - Architecture reviews;
  - Security reviews;
  - Final release reviews;
  - Full-stack integration reviews;
  - Delivery readiness reviews.

## Review report format

Review findings should include:

- **Severity**: Blocker | Important | Optional
- **Location**: File, line, or symbol
- **Explanation**: Context and impact
- **Minimal recommended correction**

AI review output is advisory; suggestions are adjudicated by Thiago before application.
